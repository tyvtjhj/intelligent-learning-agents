# 参与贡献 Contributing

感谢你愿意让 Feynman Tutor 更好用！这个技能的质量，几乎完全取决于**类比是否贴切、图是否清晰、术语是否真的讲透**。所以最有价值的贡献往往不是代码，而是教学法本身的打磨。

## 你可以贡献什么

- **更好的类比** —— 某个概念你想到了一个更一针见血的生活比喻？非常欢迎。
- **更清晰的像素图** —— 把抽象机制画得更直观的 ASCII 图。
- **新的测试用例** —— 尤其是「近似坑」：看着该触发、其实不该触发（或反过来）的真实提问。它们能帮我们校准触发描述。
- **错误修正** —— 发现技能里某段解释其实讲错了 / 讲歪了。
- **文档改进** —— README、示例、说明的任何优化。

## 怎么提交

1. Fork 本仓库
2. 新建分支：`git checkout -b improve/better-dns-analogy`
3. 修改 `SKILL.md` / 文档
4. 提交时写清楚**为什么**这样改更好（这正是本技能的精神）
5. 发起 Pull Request，在描述里说明改动动机

## 风格约定

- 改 `SKILL.md` 时，**保留「为什么这样做」的解释**——这是技能能灵活发挥的关键，不要把它压缩成生硬的命令。
- 教学示例统一用中文、零基础口吻。
- 像素图请在等宽字体下对齐检查。

## 报告问题

发现技能「该触发时没触发」「不该触发时乱触发」，或某段讲解有误，欢迎开 Issue，附上你的真实提问原文，这对优化触发描述特别有帮助。
