# 教学示例 · Example Session

> 这是 **Feynman Tutor** 启用后的真实风格示例。用户想搞懂「网络分层」，技能用「概念地图 → 像素图 → 费曼讲解 → 术语账本 → 菜单」的节奏招呼他。

---

**用户：** 帮我理解一下，为什么网上说的「分层」是网络的基础？我基础为零。

**Feynman Tutor：**

好，我们从一个总地图开始。整个网络世界的基石就一句话：

> **每一层只干一件事，只跟上下相邻的一层打交道，完全不关心更远的层在干嘛。**

### 🗺️ 先看骨架

```
┌─────────────────────────────────────────────────┐
│ 应用层   "这串数据是什么意思"   ← 网页/视频       │
│ 传输层   "送给哪个程序"         ← 端口号 :443     │
│ 网络层   "怎么跨全世界找到它"   ← IP 地址         │
│ 链路层   "本地这一步交给谁"     ← MAC 地址        │
│ 物理层   "怎么把1变成能动的东西" ← 光/电/电磁波    │
└─────────────────────────────────────────────────┘
       发送 = 从上往下层层套信封（封装）
       接收 = 从下往上层层拆信封（解封装）
```

### 🧩 用「套娃」看懂它

费曼一下：你发数据时，从上往下，每层在外面**多包一层信封**；对方收到时，从下往上，每层**拆掉自己那层**。

```
   ┌─────────────────────────────────────────────┐
   │ 应用层  [HTTP]      "把首页给我"               │  ← 内容本身
   │ 传输层  [TCP] ┃     "把首页给我"               │  ← 贴上"给443号房间"
   │ 网络层  [IP]  ┃ TCP ┃ "把首页给我"            │  ← 贴上"寄往全球地址"
   │ 链路层  [MAC] ┃ IP ┃ TCP ┃ "把首页给我"       │  ← 贴上"下一步交给谁"
   │ 物理层  1011010...（光/电/电磁波）             │  ← 整个变成 0/1 信号
   └─────────────────────────────────────────────┘
```

关键点：**每层只读自己那层的信封，不拆里面的。** 路由器是「网络层设备」，它只看 IP 那层信封决定往哪转，**根本不拆开看你 HTTP 里写了什么**——这就是为什么一台路由器能同时转发你的网页、视频、游戏：它一视同仁，只认 IP。

记住这张套娃图，因为**「梯子 / 代理」的全部魔法，就是在某几层把信封偷偷换掉**。

### 📒 术语账本（本次新增）

| 简写 | 全称 | 一句话费曼 | 所属层 |
|---|---|---|---|
| 🆕 IP | Internet Protocol | 全球通用的「门牌地址」 | 网络层 |
| 🆕 MAC | Media Access Control | 设备出厂烧死的「硬件指纹」 | 链路层 |
| 🆕 HTTP | HyperText Transfer Protocol | 浏览器和网站对话的固定句式 | 应用层 |
| 🆕 TCP | Transmission Control Protocol | 挂号信+回执，丢了重发 | 传输层 |
| 🆕 端口 | Port | 机器里的「房间号」，区分数据给哪个程序 | 传输层 |

### 🧭 接下来想钻哪个？

- **A.** 跟一个数据包走完整全程（按回车→DNS→握手→加密→拿到网页）
- **B.** 单独把「IP 和 MAC 到底啥区别」掰开揉碎
- **C.** 把「梯子」放进这张套娃图，看它精确改了第几层
- **D.** 进入物理层，看一个「1」怎么变成光在玻璃里跑
