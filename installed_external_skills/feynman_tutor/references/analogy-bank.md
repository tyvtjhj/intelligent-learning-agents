# 类比库 · Analogy Bank

> 纪律 1「费曼化」的弹药库。讲到这些概念时，优先从这里取一个**已验证好用**的类比；但**不要生搬**——如果对当前用户有更贴切的比喻，临场造一个更好的。类比的标准是:**能把陌生概念挂到用户已有的日常经验上**。

## 怎么用
- 一个概念先给类比(画面),再回到术语。
- 同一个概念可能有多个类比,挑最贴合用户背景的那个。
- 类比有边界,讲完顺手点一句"这个比喻在哪会失效",防止用户过度外推。

## 计算机 / 网络

| 概念 | 类比 | 一句话 |
|------|------|--------|
| DNS(域名系统) | 电话簿 | 把记得住的名字翻成机器用的数字地址 |
| IP 地址 | 门牌地址 | 全球唯一、写在信封上的最终住址 |
| MAC 地址 | 当面交接的那张脸 | 眼前这一步把东西递给谁 |
| 端口 Port | 大楼里的房间号 | 同一地址下区分给哪个程序 |
| TCP | 挂号信 + 回执 | 每件确认收到、丢了重发、保证顺序 |
| UDP | 大喇叭喊话 | 发出去不管收没收到，图快 |
| 三次握手 | 打电话前"喂能听到吗/能你呢/能开始吧" | 确认双向都通才开口 |
| TLS / HTTPS | 给对话套加密信封 | 中间人截到只看到乱码 |
| 数字证书 | 公安局发的身份证 | 证明对方真是它本人，防冒充 |
| NAT | 公司前台总机分机 | 全楼共用一个对外号码，前台记账转接 |
| 路由 / 转发 | 快递分拣中心层层转手 | 每站只往更近的方向递，无人知全程 |
| 代理 / VPN | 让别人替你去拿、再转交给你 | 改写目的地 + 套一层隧道 |
| 缓存 Cache | 把常用的东西放手边抽屉 | 不用每次都跑去仓库取 |
| 带宽 vs 延迟 | 水管粗细 vs 来回跑一趟的时间 | 粗管子≠跑得快，看你要传大文件还是要快响应 |

## 操作系统 / 编程

| 概念 | 类比 | 一句话 |
|------|------|--------|
| 进程 vs 线程 | 工厂(进程)里的工人(线程) | 工人共享工厂的资源 |
| 命名空间隔离 | 给每个房客单独的水电表和门牌 | 互相看不见、抢不到对方的东西 |
| 栈 vs 堆 | 自助餐叠盘子(栈) vs 大仓库领号取货(堆) | 栈后进先出、自动；堆手动、灵活 |
| 递归 | 俄罗斯套娃 / 两面镜子对照 | 自己调用自己，直到最小一层 |
| 指针 | 写着"东西在几号柜"的纸条 | 纸条不是东西本身，是它的位置 |
| 编译 vs 解释 | 整本翻译好再读 vs 边读边口译 | 一个先全转、一个逐句转 |
| 异步 / 回调 | 餐厅取餐叫号 | 点完先去坐，好了再叫你，不用站着干等 |

## 机器学习 / AI

| 概念 | 类比 | 一句话 |
|------|------|--------|
| 神经网络训练 | 反复调音的旋钮 | 错了就微调每个旋钮，慢慢逼近正确 |
| 反向传播 | 追责到每个环节 | 从最终错误倒推，每个旋钮该担多少责 |
| 梯度下降 | 蒙眼下山，专挑最陡的方向迈步 | 沿最快下坡方向一点点走到谷底 |
| 过拟合 | 死记答案的学生 | 考过的全对，没见过的全错 |
| 嵌入 Embedding | 给每个词在地图上标个坐标 | 意思近的词，坐标也近 |
| 注意力 Attention | 读句子时眼睛重点盯哪几个词 | 按相关度分配"看的力度" |

## 物理 / 数学

| 概念 | 类比 | 一句话 |
|------|------|--------|
| 傅里叶变换 | 把一段和弦拆回单个音符 | 任何波都能拆成一堆纯正弦波之和 |
| 熵 | 房间的乱度 | 不收拾只会越来越乱，要花能量去整理 |
| 微分 | 此刻的瞬时速度 | 放大到无穷小看变化率 |
| 积分 | 把无数小薄片累加起来 | 求面积/总量 = 切片再求和 |
| 概率分布 | 飞镖落点的密集程度图 | 哪里落得多，哪里概率高 |

## 金融 / 经济

| 概念 | 类比 | 一句话 |
|------|------|--------|
| 复利 | 滚雪球 | 利息再生利息，越滚越快 |
| 期权 | 不强制履约的"预订权" | 付点定金锁价，到时划算就要、不划算就弃 |
| 通货膨胀 | 同样的钱、变小的购物袋 | 钱没变，能买的东西变少 |
| 对冲 | 买保险 | 多花一点，换"反方向出事时有人赔" |
| 流动性 | 东西好不好立刻变现 | 现金最流动，房子最不流动 |

> 这个库会随使用增长。发现某个类比讲完用户秒懂，就把它补进来;发现某个类比常引起误解，就标注它的失效边界或换掉。
