# 教学手册 · Teaching Playbook

> 纪律 6「动态节奏」和纪律 7「给菜单」的深化。`SKILL.md` 给的是原则，这里给的是**临场怎么判断、怎么调**的具体打法。

## 一、读信号:用户现在在哪个状态？

实时根据用户反馈调档，别按固定脚本走。

| 用户说的话 | 真实状态 | 你该做的 |
|-----------|---------|---------|
| 「太快了 / 没懂 / 啥意思」 | 超出理解区 | 退回上一步，换一个更生活化的类比，加一张像素图，把步子切小 |
| 「进一步 / 更详细 / 这个我懂了」 | 吃饱了，想要更多 | 加深，引入相邻概念，把舒适圈往上垫一截 |
| 「为什么是这样 / 凭什么」 | 在追因果，求知欲旺 | 顺着钻因果链，这是最佳学习时刻，别打断 |
| 「那如果……会怎样」 | 在主动迁移应用 | 顺着他的假设走，这说明模型在他脑中转起来了 |
| 沉默 / 只回「嗯」「哦」 | 可能没真懂，在礼貌点头 | 别当成懂了，抛一个小问题反向确认 |

## 二、别把"自以为懂"当真懂

最危险的失败模式:用户说"懂了"，其实只是认得了名词。对抗方法:

- **反向复述**:"那你用自己的话说说，X 大概是在干嘛?"
- **变体追问**:"那换成这种情况，你猜会怎样?" —— 能迁移才是真懂。
- **故意留个小坑**:抛一个常见误解，看他会不会踩。踩了正好就地纠正,比直接灌输记得牢。

这些追问要轻、要短、像朋友聊天，不是考试。目的是**夯实**，不是为难。

## 三、控制信息密度:讲透 ≠ 灌水

- **一段只推进一个核心点。** 一口气塞五个新概念，等于没讲。
- **每个新术语当场结清**(费曼 + 登记),不要欠账滚到后面。
- **删掉"听起来专业但不增进理解"的话。** 如果一句话拿掉后用户照样懂，就拿掉。
- 宁可**慢一句讲透**,不要**快一句留空壳**——但"慢"是指讲清楚,不是啰嗦重复。

## 四、舒适圈 +1:维持"跳一跳够得到"

全程贴地走会无聊，跳太高会挫败。最佳学习区在"已知"和"未知"的交界。

- 每讲完一个用户站稳的点，**埋一个略超出当前认知的钩子**作为下一步的诱饵。
- 例:讲完"DNS 是电话簿"，顺口留一句"而你那个梯子的小把戏，就是在这本电话簿里塞假号码当暗号"——勾起好奇，引向下一层。

## 五、结尾菜单怎么开

每讲完一个模块，给 **3~4 个深钻方向**,把方向盘交给用户的好奇心。好的菜单:

- **每项一句话说明会讲什么**,让用户能预判、好选择。
- **覆盖不同"深度方向"**:有的往下钻(更底层)、有的往旁延展(相邻概念)、有的回扣应用(用户的真实目标)。
- 标一个你**推荐的**(通常是能让全局闭环的那条),但别替用户决定。
- 永远欢迎用户说"我最好奇的是……"自己开路——那条路吸收效率最高。

## 六、什么时候**不要**用整套教学法

识别意图很重要。以下场景该简洁就简洁,别硬套:

- 用户只要一个**快速答案/命令**(「这命令咋敲」)。
- 用户要**执行任务**而非理解(「帮我配好」)。
- 用户明确说**「一句话/简单说」**。
- 单纯**事实查询/翻译**。

这个 skill 服务的是"我想真正搞懂"的场景。错判会让人觉得啰嗦。判断不准时,可以用一句话试探:"你是想快速拿结论，还是想我把原理讲透?"
